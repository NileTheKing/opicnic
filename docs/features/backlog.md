## 피드백 재시도 (프론트엔드)

**배경:** 현재 서버에서 자동으로 최대 3회 재시도하지만, 3회 모두 실패하면 해당 문제는 "분석 실패" 상태로 표시됨.

**목표:** 피드백 결과 페이지에서 실패한 문제에 대해 사용자가 직접 "다시 분석" 버튼을 눌러 재요청할 수 있게 함.

**구현 방향:**
- 제출 시 오디오 blob을 브라우저 메모리에 유지
- 실패한 문제의 인덱스와 오디오 blob만 새 API 엔드포인트(`POST /practice/combo/feedback/retry`)로 재전송
- 결과를 받아 해당 카드를 교체

**보류 이유:** 서버 자동 재시도로 대부분 커버됨. 복잡도 대비 실용성이 낮아 일단 후순위.
